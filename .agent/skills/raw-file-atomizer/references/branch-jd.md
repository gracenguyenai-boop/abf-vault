# Branch JD — Job Descriptions của 5 Nhánh

> Dùng cho Phase 1: BRANCH MATCH.
> Đọc INTENT từ Phase 0, so sánh với JD từng nhánh bên dưới.
> KHÔNG đếm keyword — đọc hiểu semantic intent rồi match với JD.

---

## CC — Content Creator

### Job Description
**Tôi lưu trữ kiến thức về NGHỀ LÀM NỘI DUNG.** Cụ thể:
- Frameworks tư duy về content (STEPPS, 3Us, PAS, AIDA, viral patterns)
- Kỹ thuật viết thực chiến (hook, body, CTA, transition, human voice)
- Quy tắc và ràng buộc của platform (TikTok policy, từ ngữ cấm, định dạng)
- Bài học cải tiến từ thực tế (PDCA, video review, script performance)

### Ownership Test
> **"Nội dung này giúp tôi VIẾT NỘI DUNG TỐT HƠN không?"**
> → YES → CC

### Exclusion Rule (KHÔNG phải của CC)
| Loại content | Đi về đâu |
|---|---|
| Thông tin thị trường (dù cần để viết content) | TD |
| Hồ sơ khách hàng cụ thể | LS |
| Script đã ship của kênh cụ thể | VJ |
| Kiến thức ngân hàng không phục vụ việc viết script | LS hoặc TD |

### Edge Case: Kỹ Năng Viết vs Kiến Thức Cross-Branch

Dùng Cross-Branch Test: "CC + LS + TD cùng đọc đều thấy hữu ích?"

| Nội dung | Branch | Lý do |
|---|---|---|
| Hook câu hỏi, viết CTA, transition TikTok | CC | LS và TD không viết script → Cross-Branch = FAIL |
| Từ ngữ cấm TikTok tài chính, hook pattern ABF | CC | Đặc thù platform/kênh → CC L3 |
| Nguyên tắc giải thích đơn giản cho người không chuyên | Shared | CC viết script, LS tư vấn KH, TD trình bày phân tích → Cross-Branch = YES |
| Storytelling cơ bản (setup-conflict-resolution) | Shared | Cả 3 đều dùng trong ngữ cảnh của mình → Cross-Branch = YES |

## LS — Loan Specialist

### Job Description
**Tôi lưu trữ kiến thức về NGHỀ XỬ LÝ HỒ SƠ TÍN DỤNG.** Cụ thể:
- Framework phân tích hồ sơ (4F, risk matrix, 5C, DTI/LTV)
- Kỹ năng thực chiến nghiệp vụ (đọc CIC, thẩm định tài sản, cấu trúc hồ sơ)
- Kiến thức đặc thù An Bình (sản phẩm, pricing, điều kiện nội bộ, quy trình phê duyệt)
- Case đã phân tích đầy đủ 4F (L4/4fcases)

### Ownership Test
> **"Nội dung này giúp tôi XỬ LÝ ĐƯỢC HỒ SƠ VAY CỤ THỂ không?"**
> → YES → LS

### Exclusion Rule (KHÔNG phải của LS)
| Loại content | Đi về đâu |
|---|---|
| Tin tức thị trường lãi suất chung | TD |
| Kỹ thuật viết script | CC |
| Script đã ship | VJ |
| Phân tích vĩ mô không liên quan đến quyết định hồ sơ cụ thể | TD |

### Edge Case: Lãi Suất An Bình vs Lãi Suất Thị Trường

| Nội dung | Branch | Lý do |
|---|---|---|
| Lãi suất sản phẩm cụ thể An Bình (kỳ hạn, pricing, điều kiện nội bộ AB) | LS | CHỈ đúng với AB → LS L3 KienThucDacThu |
| Lãi suất NHNN, ngân hàng thương mại nói chung, xu hướng thị trường | TD | Thế giới bên ngoài → TD domain TC |
| Lãi suất VPBank, Vietcombank (ngân hàng đối thủ, không phải AB) | TD | Không đặc thù AB → TD domain TC |

### Edge Case: Văn Bản Quy Phạm vs Triển Khai Nội Bộ AB

| Nội dung | Branch | Lý do |
|---|---|---|
| Nghị định / Thông tư NHNN thay đổi quy định cho vay (áp dụng toàn ngành) | TD / CS | Luật chung toàn ngành → TD Ownership Test = YES |
| AB điều chỉnh quy trình nội bộ / sản phẩm theo quy định mới | LS / L3 | Đặc thù AB implementation → LS L3 "CHỈ ĐÚNG với AB" |
| File raw chứa cả 2 góc (luật chung + AB triển khai) | SPLIT | Tách thành TD/CS note + LS/L3 note |

## TD — Trà Đá / Market Analyst

### Job Description
**Tôi lưu trữ kiến thức về MÔI TRƯỜNG KINH TẾ BÊN NGOÀI.** Cụ thể:
- Tin tức và sự kiện kinh tế (NHNN, lãi suất, BĐS, CK, luật mới)
- Phân tích vĩ mô và xu hướng thị trường
- Quan sát phi hiển nhiên về môi trường tài chính
- Quan điểm chuyên gia và dự báo thị trường

### Ownership Test
> **"Nội dung này mô tả THẾ GIỚI BÊN NGOÀI ảnh hưởng đến quyết định vay/đầu tư của khách không?"**
> → YES → TD

### Exclusion Rule (KHÔNG phải của TD)
| Loại content | Đi về đâu |
|---|---|
| Cách xử lý hồ sơ tín dụng cụ thể | LS |
| Kỹ thuật viết content | CC |
| Script đã tạo ra | VJ |

## Shared — Cross-branch Foundation

### Job Description
**Tôi lưu trữ KIẾN THỨC NỀN TẢNG mà toàn bộ hệ thống cần.** Đây là "ngôn ngữ chung" — những khái niệm, frameworks cơ bản mà nếu thiếu thì CC không viết được, LS không phân tích được, TD không hiểu được bối cảnh.

### Ownership Test
> **"Nếu xóa note này, cả CC lẫn LS lẫn TD đều bị thiếu — không phải chỉ một nhánh?"**
> → YES → Shared

### Cross-Branch Test (bắt buộc thêm)
> "Nếu một người CC và một người LS và một người TD cùng đọc note này, cả 3 đều thấy hữu ích không?"
> → YES → Shared | Chỉ 1-2 người thấy hữu ích → về đúng nhánh của họ

### Exclusion Rule (KHÔNG phải của Shared)
| Loại content | Đi về đâu |
|---|---|
| Kiến thức chỉ dùng cho 1 nhánh | Về đúng nhánh đó |
| Nội dung chưa phân loại được | Inbox tương ứng (không phải Shared) |

## VJ — Virtual Journalist (per-channel)

### Job Description
**Tôi lưu trữ kiến thức và output GẮN VỚI MỘT KÊNH TIKTOK CỤ THỂ.** Cụ thể:
- DNA giọng nói của kênh đó (xưng hô, tone, style riêng biệt)
- Kịch bản đã ship và insight từ hiệu suất thực tế
- Bất kỳ thứ gì CHỈ đúng với kênh này, không dùng được kênh khác

### 4 kênh hiện có
| Kênh | Folder |
|---|---|
| VJ-AnBinh | `VJ/VJ-AnBinh/` |
| VJ-Dat | `VJ/VJ-Dat/` |
| VJ-Khang | `VJ/VJ-Khang/` |
| VJ-Thuy | `VJ/VJ-Thuy/` |

### Ownership Test
> **"Nội dung này chỉ có ý nghĩa khi gắn với kênh [tên] cụ thể không?"**
> → YES → VJ-[tên]

### Exclusion Rule (KHÔNG phải của VJ)
| Loại content | Đi về đâu |
|---|---|
| Kiến thức content chung (không gắn kênh) | CC |
| Kiến thức ngân hàng chung | LS hoặc TD |

## Cơ chế quyết định Branch

```
BƯỚC 1: Đọc INTENT từ Phase 0
BƯỚC 2: Với mỗi nhánh, trả lời Ownership Test (YES / NO / PARTIAL)
BƯỚC 3:
  - Đúng 1 YES → nhánh đó thắng
  - Nhiều YES → chọn nhánh JD mô tả INTENT gần nhất (bắt buộc giải thích lý do)
  - Không YES nào rõ → Confidence < 70% → HỎI USER
BƯỚC 4: Kiểm tra Exclusion Rule của nhánh thắng
  → Nếu content bị loại trừ → reconsider
```

## Ví dụ quyết định

| INTENT | Branch thắng | Lý do |
|---|---|---|
| "NHNN điều chỉnh lãi suất tháng 5/2026. Hữu ích cho TD vì giúp hiểu bối cảnh vĩ mô." | TD | Ownership Test TD = YES; LS = PARTIAL (liên quan nhưng không phải hồ sơ cụ thể) |
| "Hướng dẫn viết hook gây tò mò. Hữu ích cho CC vì dạy kỹ năng viết cụ thể." | CC | Ownership Test CC = YES rõ ràng |
| "Hồ sơ vay 2.5 tỷ của KH X. Hữu ích cho LS vì là dữ liệu phân tích case thực tế." | LS | Ownership Test LS = YES; có thể dùng cho VJ nhưng gắn kênh chưa rõ |
| "Định nghĩa DTI và cách tính. Hữu ích cho cả CC, LS, TD." | Shared | Cross-Branch Test = YES cả 3 |
| "Script kênh VJ-Khang tập 15 về vay mua nhà. Chỉ có ý nghĩa với kênh này." | VJ-Khang | Ownership Test VJ = YES; gắn kênh rõ ràng |
